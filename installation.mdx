# Installation

This guide will walk you through setting up your development environment for Dioxus, including installing Rust, the Dioxus CLI tool, and platform-specific dependencies.

## Prerequisites

Before you begin, ensure you have:

- A code editor (VS Code, RustRover, or your preferred editor)
- Terminal/command line access
- Internet connection for downloading dependencies

<Note>
The minimum supported Rust version for Dioxus is **1.85.0**. The installation steps below will install the latest stable version.
</Note>

## Step 1: Install Rust

Dioxus requires Rust to be installed on your system. If you don't have Rust installed yet, follow these instructions:

<Tabs>
  <Tab title="macOS / Linux">
    Open your terminal and run:

    ```bash
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    ```

    Follow the on-screen instructions. When prompted, choose option 1 for the default installation.

    After installation completes, configure your current shell:

    ```bash
    source $HOME/.cargo/env
    ```

    Verify the installation:

    ```bash
    rustc --version
    cargo --version
    ```
  </Tab>

  <Tab title="Windows">
    1. Download and run [rustup-init.exe](https://win.rustup.rs/)
    2. Follow the installation wizard, choosing the default options
    3. Restart your terminal or command prompt

    Verify the installation by opening a new terminal and running:

    ```powershell
    rustc --version
    cargo --version
    ```

    <Warning>
    On Windows, you'll also need the Microsoft C++ Build Tools. The Rust installer will guide you to install them if they're not already present.
    </Warning>
  </Tab>
</Tabs>

## Step 2: Install cargo-binstall (Recommended)

`cargo-binstall` allows you to install pre-compiled binaries of Rust tools, making the installation much faster than building from source.

```bash
cargo install cargo-binstall
```

<Tip>
If you skip this step, you can still install the Dioxus CLI directly with `cargo install dioxus-cli`, but it will take longer to compile.
</Tip>

## Step 3: Install Dioxus CLI

The Dioxus CLI (`dx`) is the command-line tool for creating, developing, and building Dioxus applications. It provides hot-reload, bundling, and deployment features.

<CodeGroup>
```bash With cargo-binstall (Recommended)
cargo binstall dioxus-cli
```

```bash From crates.io
cargo install dioxus-cli
```

```bash From latest Git (Bleeding edge)
cargo install --git https://github.com/DioxusLabs/dioxus dioxus-cli --locked
```
</CodeGroup>

Verify the installation:

```bash
dx --version
```

You should see output like:
```
dioxus-cli 0.7.0
```

## Step 4: Platform-Specific Setup

Depending on which platforms you want to target, you may need additional dependencies.

### Web Development

For web development, you need to add the WebAssembly target:

```bash
rustup target add wasm32-unknown-unknown
```

<Note>
No additional tools are required for web development - the Dioxus CLI includes everything you need!
</Note>

### Desktop Development

Desktop apps use the system's webview. Here's what you need for each platform:

<Tabs>
  <Tab title="macOS">
    No additional dependencies required! macOS includes WebKit by default.

    <Tip>
    For building macOS app bundles (.app), the CLI uses built-in tools.
    </Tip>
  </Tab>

  <Tab title="Linux">
    Install WebKitGTK and other dependencies:

    <CodeGroup>
    ```bash Ubuntu/Debian
    sudo apt install libwebkit2gtk-4.1-dev \
      libgtk-3-dev \
      libayatana-appindicator3-dev \
      libssl-dev
    ```

    ```bash Fedora
    sudo dnf install gtk3-devel \
      webkit2gtk4.1-devel \
      openssl-devel
    ```

    ```bash Arch Linux
    sudo pacman -S webkit2gtk-4.1 \
      gtk3 \
      libayatana-appindicator
    ```
    </CodeGroup>
  </Tab>

  <Tab title="Windows">
    No additional dependencies required! Windows uses the built-in WebView2.

    <Note>
    Windows 10/11 includes WebView2 by default. For older systems, it will be automatically installed when your app runs.
    </Note>
  </Tab>
</Tabs>

### Mobile Development

Mobile development requires platform-specific SDKs and tools.

<Tabs>
  <Tab title="Android">
    <Steps>
      <Step title="Install Android Studio">
        Download and install [Android Studio](https://developer.android.com/studio) which includes the Android SDK.
      </Step>

      <Step title="Install NDK">
        Open Android Studio → Tools → SDK Manager → SDK Tools tab
        
        Check and install:
        - Android SDK Platform-Tools
        - Android NDK
      </Step>

      <Step title="Add Rust targets">
        ```bash
        rustup target add aarch64-linux-android
        rustup target add armv7-linux-androideabi
        rustup target add i686-linux-android
        rustup target add x86_64-linux-android
        ```
      </Step>

      <Step title="Set environment variables">
        Add to your shell profile (~/.bashrc, ~/.zshrc, etc.):

        ```bash
        export ANDROID_HOME=$HOME/Android/Sdk
        export NDK_HOME=$ANDROID_HOME/ndk/<version>
        ```

        Replace `<version>` with your installed NDK version.
      </Step>
    </Steps>
  </Tab>

  <Tab title="iOS">
    <Steps>
      <Step title="Install Xcode">
        Download and install [Xcode](https://apps.apple.com/us/app/xcode/id497799835) from the Mac App Store.
        
        After installation, open Xcode to accept the license agreement.
      </Step>

      <Step title="Install Xcode Command Line Tools">
        ```bash
        xcode-select --install
        ```
      </Step>

      <Step title="Add Rust targets">
        ```bash
        rustup target add aarch64-apple-ios
        rustup target add aarch64-apple-ios-sim
        rustup target add x86_64-apple-ios
        ```
      </Step>
    </Steps>

    <Note>
    iOS development requires a Mac with macOS. You'll also need an Apple Developer account for deploying to physical devices.
    </Note>
  </Tab>
</Tabs>

## Step 5: Editor Setup (Optional but Recommended)

While you can use any text editor with Dioxus, these extensions will improve your development experience:

### VS Code

Install these extensions:

1. **rust-analyzer** - Rust language support with autocomplete, diagnostics, and more
2. **Dioxus** - Syntax highlighting and autocompletion for RSX

You can install them from the VS Code marketplace or via command line:

```bash
code --install-extension rust-lang.rust-analyzer
code --install-extension dioxuslabs.dioxus
```

<Tip>
The Dioxus extension provides RSX autocompletion, formatting, and component navigation!
</Tip>

### RustRover / IntelliJ IDEA

1. Install the **Rust** plugin from JetBrains Marketplace
2. Install the **Dioxus** plugin for RSX support

## Verification

Let's verify your installation is complete by creating a test project:

<Steps>
  <Step title="Create a new project">
    ```bash
    dx new my-test-app
    cd my-test-app
    ```

    When prompted, select your preferred platform (web, desktop, or fullstack).
  </Step>

  <Step title="Run the development server">
    ```bash
    dx serve
    ```

    This should compile your app and open it in your default browser (for web) or launch a window (for desktop).
  </Step>

  <Step title="Test hot-reload">
    Open `src/main.rs` in your editor and change some text in the `rsx!` macro. Save the file and watch your app update instantly!
  </Step>
</Steps>

<Success>
If you see your app running and hot-reload is working, congratulations! Your development environment is ready.
</Success>

## Troubleshooting

<AccordionGroup>
  <Accordion title="'dx' command not found">
    Make sure `~/.cargo/bin` is in your PATH:

    ```bash
    echo $PATH
    ```

    If it's not there, add this to your shell profile:

    ```bash
    export PATH="$HOME/.cargo/bin:$PATH"
    ```

    Then restart your terminal or run `source ~/.bashrc` (or `~/.zshrc`).
  </Accordion>

  <Accordion title="Compilation errors on Linux">
    If you see errors about missing libraries, ensure you've installed all the development dependencies listed in the Desktop Development → Linux section above.

    Common missing packages:
    - `pkg-config`
    - `libssl-dev` or `openssl-devel`
    - `webkit2gtk` libraries
  </Accordion>

  <Accordion title="Slow initial compilation">
    The first time you build a Dioxus app, Rust needs to compile all dependencies. This can take several minutes.

    Subsequent builds will be much faster thanks to incremental compilation and caching.

    To speed up compilation:
    - Use `cargo-binstall` for pre-compiled CLI tools
    - Consider using `sccache` for distributed compilation caching
    - Use `mold` or `lld` as a faster linker
  </Accordion>

  <Accordion title="WebView2 issues on Windows">
    If your desktop app doesn't launch on Windows:

    1. Ensure Windows is up to date
    2. Download and install [WebView2 Runtime](https://developer.microsoft.com/en-us/microsoft-edge/webview2/) manually
    3. Restart your application
  </Accordion>
</AccordionGroup>

## Keeping Up to Date

To update Dioxus CLI to the latest version:

```bash
cargo binstall dioxus-cli --force
```

To update Rust itself:

```bash
rustup update
```
